Javascript has an advanced mode! For special people who know about it! Not really of course, but it's a fun sentiment. Evidently as of es5 javascript broke some stuff with backwards compatibility. I'm sure this made many people extremely happy and was settled as peacefully as every other holy war in the history of Earth. Regardless, in order to unbreak the old stuff, the new stuff was disabled by default, which means we have to enable it when we want to use it. we do this using

February 19

Not a lot of blog in the past few weeks due to an influx of homework and social obligations, but I finally got a chance to work today! Did a few things actually.

  • Updated my dotfiles repo on github to match my new setup, loving how easy vs code makes it to back everything up to the repo and move to a new system if needed. Also, since I don't think I've said it here: bonsai.sh is far and away my favorite bit of software I've used recently. I love the way it integrates with neofetch, clean and pleasing every time I open a terminal. 10/10 would recommend.

  • Worked on my networking homework. We're using mininet to simulate various network topologies, and I struggled to get it running properly on my home server. Both the source and packaged installs gave me weird errors, which I traced back to openvswitch but couldn't remedy. I ended up just using mininet's prebuilt vm image with qemu to run the tests our professor requested.

  • Started my senior design project! My team is working on a scheduling app for a music institute in Dallas using Google OR Tools for optimization. I'll be working with another teammate on the front end of the project, and I'm really looking forward to digging in and making something useful.

    We decided to use Vue as a front end framework for the app since a) doing input and output by hand for the kinds of data we're going to be working with would be extremely taxing, and b) working with components will allow for a clean, consistent experience that can scale to whatever the project needs. I personally wanted to find c) a front end framework that I could really get into. While I might have preferred React for its wider industry adoption and, by extension, job prospects, I think Vue is the better choice for the scale of project we're targeting. Of course, if we find we need the power of React or another more comprehensive platform, switching over should be a fairly simple, and educational, experience.
Later today I'll finish my machine learning assignment in R and hopefully get to make a little more progress on that js tutorial.
